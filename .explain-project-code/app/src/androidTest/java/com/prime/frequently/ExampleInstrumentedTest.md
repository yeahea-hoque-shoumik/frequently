## Class: com.prime.frequently.ExampleInstrumentedTest

**Summary**
A scaffold instrumented test class generated by Android Studio when the project is created. It runs directly on an Android device (not the JVM) and verifies that the app's package name matches the expected value — acting as a sanity check that the test runner and instrumentation are wired up correctly.

_No instance variables._

### `useAppContext`

**Function Name:** `com.prime.frequently.ExampleInstrumentedTest.useAppContext`

**What** — Retrieves the target app's `Context` via instrumentation and asserts that its package name equals `"com.prime.frequently"`.

**Why** — Serves as the minimal proof that the instrumented test environment is functional: if the `Context` is retrievable and the package name matches, the test runner is correctly targeting this app's process.

**How**
1. Calls `InstrumentationRegistry.getInstrumentation().targetContext` to obtain the `Context` of the app under test
2. Calls `assertEquals` to compare the actual package name against the expected string `"com.prime.frequently"`
3. Fails the test if they differ, passing otherwise

**Insights**
- This is boilerplate generated by Android Studio — not a meaningful business test
- `@RunWith(AndroidJUnit4::class)` is required to run on-device; without it the test would be a plain JUnit test targeting the JVM only
- A good starting point to verify the instrumented test infrastructure before writing real UI or integration tests
